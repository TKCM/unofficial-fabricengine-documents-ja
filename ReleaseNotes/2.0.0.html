Fabric Engine 2 リリースノート¶

Fabric Engine version 2.0.0
2015年9月30日 リリース
Copyright 2010-2015 Fabric Software Inc. All rights reserved.

Fabric 2 では、簡単に強力なビジュアル・プログラミング・インターフェースを利用できる Fabric Canvas が導入されました。Fabric の Core 機能を、この全くあたらしいデータフローグラフから利用できます。Fabric for Maya、Fabric for Softimage 統合版を使えば、あなたの好みのDCCツールから ―さらには異なるDCCツール間ですら― Fabricを利用できます。

  • Fabric 2 はWindows、Linux上で動作します。OS X 向けのビルドも提供されますが、公式にサポートされているわけではありません。今後のリリースによりサポート予定です。

  • Fabric for Maya 統合版は Autodesk Maya 2014、2015、2016（SP版含むWindows/Linux版）をサポートします。OS X は今後サポート予定です。

  • Fabric for Softimage 統合版は Autodesk Softimage 2014、2015（SP版含むWindows版）をサポートします。

Fabric 2 GPU supportは NVIDIAハードウェアから利用可能です。詳細はGPUコンピューティングガイドを御覧ください。

Fabric Canvasについてのより詳細な情報はこちらから：

  • Canvas ユーザガイド
  • Canvas Programmer Guide
  • Fabric Engine 入門

ダウンロード¶

Fabric Engine 2 を取得するには http://dist.fabric-engine.com/FabricEngine/2.0.0/ から行えます。インストールについては Fabric Engine 入門 を参照してください。

Support¶

Fabric 2 は評価版、スタジオ版、教育版が無償で利用可能です。また商用契約に基づく利用も可能です。Fabric 2のベータプログラムは終了いたしました。サポートについての情報は、商用ライセンスをご利用のお客様には、私達よりご連絡しています。非商用のお客様はサポートについて以下のオプションをとることをお勧めいたします。

  • http://answers.fabricengine.com Fabric コミュニティによる Ask & answer questions。サポートに関する問題を投稿する場です。

  • https://groups.google.com/forum/#!forum/fabricengine より濃いディスカッションフォーラムです。Canvas グラフをシェアしましょう。

  • bugs@fabricengine.com もしバグを発見したら、明快な再現手段を付しEメールをください。(使用OSとDCCアプリ、それぞれのバージョンについての詳細も)Fabricからの返答は、Fabricプラットフォームの問題の解決にあたって追加情報を必要とする場合のみですので、ご了承ください。

既知の不具合、制限事項、回避策¶

注釈

Fabric 2 Beta では、Canvasグラフは拡張子 .dfg.json として保存されていましたが、.canvas に変更されました。旧式のグラフでも動作しますが、”File –> Open Graph” からは 新しい .canvas 拡張子のものしか表示されません。古いものは .canvas に変更してください。 [FE-5289]

注釈

新しい ‘persist value’ オプションがグラフのトップメニューに追加されました。以前のグラフを読み込むとこれはディフォルトで ‘false’ に設定されます。このオプションは該当ポートを右クリックし ‘Edit Port’ -> ‘metadata’ -> ‘persist value’ を選ぶことで有効化されます。それ以外の場合、再保存されてもポートの値は persist ―保持、永続化されません。ただし、MayaやSoftimage 上のグラフの場合、値はネイティブアトリビュートとして、グラフ内ではなくシーンの一部として保持されます。[FE-5194]

注釈

SpliceStandalone ―Fabric Engine 2.0 以前のもの― は Fabric 2には含まれなくなりました。もし SpliceStandalone 向けのツールを作成されていたら、書きなおす必要があります。APIの大半は似ているため、いくつかFabric 2向けに手直しを加得る必要はありますが難しくはありません。[FE-4599]

  • 不具合: Maya 2016 では 2.0.0βで作成された PolygonMesh 出力ポートをもつシーンを開くとクラッシュします。[FE-5367] これは該当アトリビュートが修正前に作成されたためです。

    回避策: 出力アトリビュートをリリースバージョン2.0.0上の PolygonMesh型で再作成します。

  • 不具合: Fabric for Maya 2014、2015使用時、Canvas 上のポートの名前をを変更する際、すぐにはノードエディタに変更が反映されない。[FE-5415]

    回避策: Canvasノードを選択し、ノードエディタの入出力ポートの接続アイコンをクリックし強制リフレッシュします。

  • 不具合: GPUコンピューティングを Maya 2016で使用すると Mayaがクラッシュ。[FE-5402]

    回避策: GPUコンピューティングを使用（ディフォルトでは使用されません）する際はMaya 2014, 2015をご利用ください。

  • 制限: サブグラフ中の read variables ノードを編集できない[FE-5236]

    回避策: 新規の read variable ノードを新しい名前で作成します。

  • 制限: Fabric for Softimage では ICEの入出力をサポートしません。[FE-5091]

    回避策: ICEとのやりとりには、旧式実装の Splice を引き続き利用できます。KL Singletons にデータを格納し、SpliceとCanvasと連携させることができます。

  • 制限: backdroup ノードや variable ノード上での右クリックから “Imploding” ―複数ノードを一纏めに連結― は行えません。[FE-4828]

    回避策: backdroups や variable ノードを選択した際は、何もないエリアを右クリックし、そのコンテキストメニューから“Implode” を選んでください。

  • 制限: KL2DFG では polymorphic nodesについて $TYPE$ 型仕様を使用します。これはノードがタイプの選択が可能であっても、そこに何型であっても接続ができてしまいます。将来のリリースではユーザビリティ向上のため、型のリストをサポートする予定です [FE-4813]

    回避策: ピン上のマウスをのせます。ツールチップ上にサポートされる型がリストアップされます。

  • 不具合: Solid Angle の SItoA Arnoldプラグインバージョン3.0.0未満を使用している場合そのままでは、Fabric for Softimage とコンフリクトします。SItoA 3.0.0以上では問題ありません。[FE-4559]

    回避策:

    • ファイルエクスプローラを開き、Fabric ワークグループフォルダを開きます。”..../FabricSoftimage2014SP2/Application/Plugins” です。

    • “tbb.dll” というファイルがあります。

    • もう一つファイルエクスプローラウィンドを開き SItoA nt-x86-64 バイナリフォルダを開きます。 ”.../SItoA/Application/Plugins/bin/nt-x86-64” です。

    • “tbb.dll” ここにも同名ファイルがありますね。

    • Fabric の方の tbb.dll ファイルを SItoA nt-x86-64 binary の SItoA’s tbb.dll に上書きコピーします。（Fabric の方にもDLLを残してください！ また上書き前にバックアップをとっておきましょう）

    • Softimage のプラグインマネージャより SItoA ワークグループが Fabric のワークグループより先に読み込まれるよう ―上に入れ替えます。

    • Softimage を再起動します。

    • Fabric Canvas のサンプルがコンフリクト無く読み込まれるようになります。

  • 不具合: プリセットに基づくグラフノードをプリセットに基づかないKL関数ノードとともに使用すると、KL関数を分離した際、プリセットからの変更点が正しくUIに反映されません。 [FE-5454]

    回避策: 変更点は一旦 Go up し、該当KL関数ノードを再編集すると確認できます。

  • 不具合: Canvas.exe を起動した直後、右サイドパネルの右クリックから Executeノードを作成するとクラッシュします。次のマイナーバージョンアップにより修正される予定です。 [FE-5400]

  • 不具合: ノードをリネームしても、Fabric Coreから付与されるユニークネームは変わりません。例:”hello”ノードを”world”にリネームしても “world”ノード編集すると、unique-to-Fabric Core のユニークネームは “hello”のままです [FE-4577]

  • 不具合: オブジェクト中、配列の辞書を使用するとメモリリークを引き起こします [FE-4404]

  • 不具合: ノードを複製すると、複製されたノードには Fabric Coreよりユニークネームが付与されます。例: “hello” を複製すると “hello_2” という名で複製されます。複製元を “world” にリネームしても複製先の名前には影響されません。ユニークネームは “hello_2”のままです [FE-5327]

  • 制限: Fabric for Maya プラグインは、アンロード不可能です。Maya起動セッション中はリロードされます [FE-5383]

  • 制限: Canvas のバリューエディタは undo/redo の結果を正しく反映しない場合があります。 [FE-5364]

  • 制限: グラフやノード名に使用できる文字は制限されます。[a-z][A-Z][0-9]が使用可能です。Fabric Coreが名前コンフリクトを検出した際、独自性を保つため、自動でアンダースコアと数字 （hoge_1）が付与されます。[FE-5285]

  • 制限: Fabric for Softimage は現時点でポートの配列をサポートしていません [FE-5088]

  • 制限: Canvas上でプリセットを保存した際、コメントは保持されません [FE-4900]

  • 制限: 一旦アトリビュートが作成されると “opaque in DCC”、”native DCC array” プロパティは変更が利きません [FE-4799]

  • 制限: DrawingHandle::drawTexturedPlane() はアルファチャンネルを正常に扱えません [FE-4792]

  • 制限: Canvas プリセットフォルダ名には ‘.’ を含むことができません [FE-4767]

  • 制限: トポロジの変更を伴うAlembic メッシュの読み込みは未対応です [FE-4535]

  • 制限: variable declaration ノードは、仮に他のノードと接続があっても、正しく評価されるためにはグラフの出力ポートに接続される必要があります。 [FE-4505]

  • 接続: FabricCoreクライアントが破棄される際、Canvas のグラフより直接アクセスされてる variable のみメモリ解放されます。グラフから触られていない ―出力ポートに接続されていない― InlineDrawing を使った描画物や Manipulation handles は生存し続けてしまいます。これをクリアするには、例えば InlineDrawing DrawingHandles であれば 手動で EmptyDrawingHandle ノードに接続します [FE-4465]

  • 不具合: Canvas バリューエディタ: OS X では描画乱れが多数発生します [FE-5129]

  • 不具合: Canvas バリューエディタ: OS X ではカラーピッカーが動作しません [FE-5458]

  • 制限: Windows 10 （のファーストリリース）には GPU の Unified Memory 周りにバグがあります。回避には NVIDIA の version 349 ドライバを使用します [FE-4381]

  • 不具合: Canvas での プリセットからの分離はドキュメントには記載されていません。 [FE-5461] その代わりここで手短に説明いたします:

    • Canvas UIより、プリセットのインスタンスであるノード内部を開きます。内部のコンテンツを確認はできますが、変更はできません。ウィンドの上部にプリセットのインスタンスであることを知らせるバーが表示されています。さらに、プリセットから分離固有化し、変更可能にするためのボタンがあります。プリセットからの分離による固有化をすると、分離後プリセットが変更されても、自動で追従更新されなくなります。意図的であれなかれ、プリセットから分離した後、分離まえの状態に戻したいのであれば、undo履歴より戻すことができます。
